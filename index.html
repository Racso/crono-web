The VM maintains an operand stack and a call stack of frames. Each frame holds its own locals, a reference to its closure, and a return address.

captured by reference, closed on exit

fig. 0.3 — garbage collector
§ 12
memory management
garbage collector
mark-and-sweep · tri-color
A tri-color mark-and-sweep GC. All heap-allocated values (strings, objects, arrays, closures) are tracked. The GC runs automatically when heap pressure exceeds a threshold.
algorithm
tri-color mark-and-sweep
trigger
heap size > threshold (default 1 MB)
roots
operand stack, globals, open upvalues
cycles
handled (back-references traced)
growth factor
2× after each full collection
The GC is stop-the-world. Concurrent collection is planned for a future revision.

FFI / DynCall
calling native code
crono-vm exposes a C API for embedding and extension. Native functions are registered with a type signature and called via DynCall, which handles ABI-level dispatch without custom stubs.
mechanism
DynCall (libdyncall)

fig. 3.3 — error model
§ 15
runtime errors
error model
throw · catch · unwind
Errors are first-class values. Any value can be thrown. The VM unwinds the call stack until a matching catch handler is found, or terminates with an uncaught error.

stack effect
description

fig. 1.4 — example: hello world
§ 17
example
hello, world
// hello.cs — crono-sharp source print("hello, world");
; compiled bytecode (disassembly) PUSH_STR "hello, world" CALL print, 1 POP HALT
The compiler emits PUSH_STR to load the string constant, CALL to invoke the built-in print function, POP to discard the return value, then HALT.
fig. 2.4 — example: fibonacci
§ 18
example — recursion
fibonacci
// fibonacci.cs int fib(int n) { if (n <= 1) return n; return fib(n - 1) + fib(n - 2); } print(fib(10));
; key bytecode (inner loop) LOAD_LOCAL 0 ; n PUSH_INT 1 LTE JMPF +3 LOAD_LOCAL 0 RET LOAD_LOCAL 0 PUSH_INT 1 SUB CALL fib, 1 LOAD_LOCAL 0 PUSH_INT 2 SUB CALL fib, 1 ADD RET
fig. 3.4 — example: native extension

// embed crono-vm in C #include "crono.h" int main() { CronoVM *vm = crono_new(); // register native sqrt crono_register_native(vm, "math.sqrt", sqrt, "d)d" ); // load and run bytecode crono_load_file(vm, "prog.cvm"); crono_run(vm); crono_free(vm); }
